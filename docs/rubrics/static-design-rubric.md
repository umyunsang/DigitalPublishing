# Static Design Rubric

Use this rubric for HTML/CSS exercises, Figma exports, screenshots, and pre-frontend layout specs.

| Category | Excellent | Needs Work |
| --- | --- | --- |
| Purpose | The viewer can immediately understand the page goal. | The page feels like elements placed without a clear message. |
| Visual hierarchy | Heading, sections, emphasis, and reading order are obvious. | Important items compete or disappear. |
| Layout | Alignment, grouping, and composition are intentional. | Blocks drift, widths are inconsistent, or grouping is unclear. |
| Spacing | Rhythm is consistent across sections and components. | Gaps feel arbitrary or crowded. |
| Typography | Font size, weight, line height, and contrast support reading. | Type scale is flat, oversized, cramped, or inconsistent. |
| Color | Palette supports structure and emphasis. | Colors are decorative only or reduce legibility. |
| Imagery | Images have clear roles, consistent crops, and useful `alt` text. | Images distort, mismatch the layout, or lack text alternatives. |
| HTML structure | Markup reflects the document structure. | Layout depends on unclear nesting or generic tags only. |
| Accessibility | Basic contrast, alt text, headings, and links are handled. | Accessibility is not considered. |
| Course fit | The solution practices the assigned layout/CSS concept. | The solution bypasses or overcomplicates the assignment goal. |

## Review Output Format

1. State the highest-impact issue first.
2. Give file and selector references when available.
3. Separate required fixes from optional polish.
4. Preserve intentional class constraints, such as fixed-width exercises.
