# Brief to Layout Method

## Required Outputs

1. Design goal
2. Content inventory
3. Layout zones
4. Grid and spacing rules
5. Typography scale
6. Color and token decisions
7. Asset requirements
8. Risks and open questions

## Static Design Bias

Keep output close to what a student can apply in plain HTML/CSS.

Use these levels:

- Sketch: rough hierarchy and page zones.
- Spec: sizes, spacing, type, colors, and image rules.
- Build notes: selectors, semantic tags, and CSS technique.

Only include implementation code when the user asks for code.
