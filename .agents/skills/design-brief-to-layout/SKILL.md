# Design Brief To Layout

## Overview

Turn a fuzzy assignment or concept into a practical static design plan. Produce design artifacts first and code only when the user asks.

## Workflow

1. Clarify the page goal, audience, content, and constraints from the brief.
2. Create a content inventory.
3. Propose page zones and reading order.
4. Define grid, spacing, typography, color, image, and token rules.
5. Identify risks such as text overflow, image crop, contrast, or fixed-width limitations.
6. Produce a layout spec or fill the local template.

## Output

Default output:

- Design goal
- Content inventory
- Layout zones
- Grid and spacing
- Typography
- Color and token decisions
- Asset list
- CSS technique recommendation
- Risks and open questions
