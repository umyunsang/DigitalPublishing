# Static Design Review

## Overview

Evaluate static design work as a course artifact. Preserve the assignment's intended CSS technique while giving concrete design, layout, and accessibility feedback.

## Workflow

1. Identify the artifact type: HTML/CSS, screenshot, Figma export, or layout spec.
2. Read the relevant files before judging the design.
3. Check whether fixed width, float, position, flex, or inline styles are intentional class topics.
4. Review hierarchy, layout, spacing, typography, color, image handling, semantics, and accessibility basics.
5. Separate required fixes from optional polish.
6. Avoid framework migration suggestions unless the user asks.

## Output

Use this shape:

- Highest-impact issue
- Required fixes
- Optional polish
- File or selector references when available
- One concise implementation hint only when useful
